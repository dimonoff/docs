# Parking Spot Configuration

By entering edit mode through the “Spatium Toolbox” search bar, you will be able to configure and modify parking spots within the corresponding zone.

<figure><img src="../../.gitbook/assets/image (46).png" alt=""><figcaption></figcaption></figure>

You can add new spots by typing “Add a Spot”. A floating menu will appear on the left side of the screen.

<figure><img src="../../.gitbook/assets/image (47).png" alt=""><figcaption></figcaption></figure>

You can add new spots by typing “Add a Spot”. A floating menu will appear on the left side of the screen.

<figure><img src="../../.gitbook/assets/image (48).png" alt=""><figcaption></figcaption></figure>



The spot will then appear in the center of the layout (as outlined in red in the image above), already selected, and a floating menu will appear on the left side of the screen to configure the following:

* Name
* Type, which will dynamically affect the icon representing the spot
* The serial number of the equipment controlling this spot, which will detect its status: This is a unique name assigned to the equipment by the manufacturer and has no character limit
* The sensor name of the equipment representing this spot to display its status
* Option to align spots vertically and horizontally

Once a spot is configured or modified, click “Apply” to save the changes, otherwise, the modification will not be recorded, then click “Save” to exit edit mode.

<figure><img src="../../.gitbook/assets/image (49).png" alt=""><figcaption></figcaption></figure>

<mark style="color:$danger;">IMPORTANT</mark>: The equipment serial number and the sensor for the equipment must belong to a sensor that is already recognized and communicating with the system. Otherwise, the configured spot(s) will remain gray, indicating that no status is detected. Green represents an available spot, and red represents an occupied spot.
